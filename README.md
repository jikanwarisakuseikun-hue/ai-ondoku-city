# 🗣️ AI音読システム Max Pro

自治体内の複数校・全クラスで同時運用が可能な、Azure AIを活用した公立小中学校向けの英語音読自動採点・提出システムです。

## 🎯 本システムの特長

1. **完全匿名による高セキュリティAI採点**
   * 児童生徒の「音声データ」と「課題英文」のみをAzure AI（Microsoft）へ送信し、発音の多角的な分析（正確さ・流暢さ・抑揚・完成度）を行います。
   * **氏名や学校名などの個人情報はAIサーバーへ一切送信されないため、情報漏洩リスクを極限まで排除しています。**
2. **学校別の「完全物理隔離」による成績管理**
   * 各校のデータは、市教委が管理する大元シートから、学校ごとに独立した専用スプレッドシートへと自動転送されます。
   * 他校の教員や児童生徒が別校のデータにアクセスすることは物理的に不可能な設計です。
3. **クラス別タブの完全自動生成（ノンプログラミング運用）**
   * 生徒が画面から提出した瞬間に、該当クラスのタブ（例: `1A`）が成績表ファイル内に自動生成されます。
   * 異動や新年度のクラス替えの際も、管理者がプログラム（GASコード）を書き換える必要は一切ありません。

## 🏗️ システム構造

【 1. 生徒の端末 】
Streamlit Cloud上のWEBアプリ画面から、音声と所属を入力して提出
│
▼ （AI解析完了後、一時的に集約）
【 2. 市教委マスターシート（大元） 】
認証用「マスタ」タブと、アプリが直接書き込む「各学校名」の受取タブで構成
│
▼ （★Google Apps Script が自動実行）
【 3. 学校別の独立成績表ファイル 】 ★各校の先生が閲覧する画面
GASがマスターシートからデータを自動回収し、学校別ファイルの中に
「1A」「1B」といった【クラス別タブ】を自動生成してデータを綺麗に格納

## 🛠️ ファイル構成

GitHubリポジトリ内には以下のファイルを配置してアプリを駆動させています。

* `app.py`: Streamlitで構築されたWebアプリ本体のPythonコード。
* `requirements.txt`: アプリの起動に必要な外部ライブラリの定義リスト。
* `README.md`: 本システムの説明書（このファイル）。

## 🔒 市教委・セキュリティ担当者向け仕様

* **個人情報の格納先**: 自治体または学校が管理するGoogle Workspace（セキュアなドメイン内）のスプレッドシート内のみ。
* **通信の安全性**: アプリとAPI間の通信はSSL/TLSにより暗号化。
* **アクセス権限**: 各学校の成績表ファイルは、該当校の担当教員のアカウントのみに「制限付き共有」を設定することで、内部不正や誤閲覧を完全に防止。
